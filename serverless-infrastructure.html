Leveraging a serverless infrastructure

I am exploring going serverless for some parts of my trading infrastructure.

What’s the situation?

I’ve been renting a cheap VPS from OVH at 3.60 EUR/month to run a fully automated intraday trading program for digital assets for a couple of years now. In August 2022, I’ve launched a Futures trading program which is currently running on my desktop computer at home. I’d like to automate it and run it in the cloud and get my life back.

My current VPS is ultra cheap and not powerful. Running multiple programs in parallel, especially running historical simulations, might crash it at times. I could rent a more expensive VPS, but I prefer to remain cheap and leverage severless functions.

Infrastructure design

Keep the VPS to run the following programs 24/7:

  • a workflow orchestrator
  • a dashboard
  • broker clients
  • market data clients

Move the signal_generator module to AWS Lambda and create a lambda function for each strategy/trading system. Use AWS S3 to access market data and save model portfolios.

Benefits

  • free up resources for the programs running 24/7 on my cheap VPS
  • infinitely scalable and parallel execution for historical simulations
  • manage both trading programs (futures and digital assets) together, and opens the possibility to add more programs
  • cheap?
